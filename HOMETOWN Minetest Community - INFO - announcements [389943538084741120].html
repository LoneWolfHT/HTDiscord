ChimneySwift 02-Jun-19 10:20 PM
ay guys, wall of text incoming
'ello folks! How 're yee doin' terday? I've finally got a passable map and mod list. If you're not interested skip the technical stuff and read the bottom, otherwise, here's why it's been so annoying to get going: === Technical Stuff === Basically, Minetest generates map in different units to how it stores and sends it. Map blocks are 16³ node blocks which are stored one per row in the map database, however map chunks are 80³ node blocks, which are what mapgen actually generates. The problem arises when the map database doesn't have a full chunk worth of blocks stored in it, as if any of the 125 map blocks within that chunk aren't already generated and stored in the database, the mapgen will run certain generators on the entire 80³ worth of map blocks, regardless of whether there are builds there or not. Since usually clients don't get sent the entire 80³ nodes worth of map for each chunk they enter, we ended up only having about 29% of the map blocks that would have been required to stop mapgen needing to "fill in" all of the saved chunks. This caused builds and stuff getting cut up by cave and rivergen, dusted with snow and overrun by trees. This is the same issue ExeterDad had, and sorted out by generating all the missing blocks and overlaying the map saves... but I was going to avoid that because I didn't want to upload a huge file to host. Thanks to Minetest not having like LITTERALLY ANY documentation on the mapgen and mapsave process (and probably mostly just me not having anywhere near as much expertise as ExeterDad has), I thought I could change some flags in the map save to fix the problem, spoiler alert: it didn't work. So with the trial and error of mapgen I finally got it working after a solid 72 hours worth of generating over 200 thousand map chunks. Turns out the final map so far is about 3.5gb uncompressed which isn't too bad. Also I got all the mods working and there shouldn't be any unknown nodes or items. === Technical Stuff ===
Anyway with that out of the way, I have a bit of a predicament. Because I had 23 different saves, and different parts of each save are of different ages, there are likely a ton of builds that aren't up-to-date on the map I have now. Also, there are a ton of awesome builds which I'd like coordinates for that I don't know of. So today I'm going to be bringing up a temporary spectate-only server with the map on it, and I want everyone to join and do 4 things: 1. Run around and remember how awesome the server was 2. Find all of the awesome builds that you or your friends made 3. Tag each build for review. I'll be setting up a really simple way to do this, you'll be able to: - Provide build info - Tell me whether or not it's up-to-date 4. Report any errors you find. For info on how to do that check this video: https://youtu.be/8LpqXbU_vU8 (basically just do /1 on one corner of the build, do /log on the opposite and then fill out the form) In order to help you find your builds, you will have access to: - A full, navigable map located at http://ht.chimneyswift.pro - Fly, fast, noclip and teleport privileges. NOTE: Just to be totally clear, this is a very temporary setup and in no way meant to be a substitute for the original HOMETOWN server, this server will be read-only (no build privilages). If the map gets too big from mapgen, I will simply delete it and restore it to how it is now. Mods will have mod perms but don't have to moderate or do anything, if you break any of the original hometown rules you'll probably just be banned. Anyone who spams the build logger with garbage builds will have all of their build logs deleted, so... don't do that. Please don't share the address or password outside of people who used to play. The final map will be available to everyone at a later date. Info: Address: ht.chimneyswift.pro Port: 30000 Default Password: hometown4days (may change) (edited)
